## Urgent Upgrade Notes 

### (No, really, you MUST read this before you upgrade)

- The exit behavior has changed. Fatal errors that previously used `klog.Fatalf` now exit with status code 1 instead of 255. ([#9778](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9778), [@anndono](https://github.com/anndono))
 
## Changes by Kind

### Feature

- ACR credential provider now supports Kubernetes Service Account (KSA) based authentication with identity bindings. Configure via flags: --ib-sni-name, --ib-apiserver-ip, --ib-default-client-id, and --ib-default-tenant-id. ([#9687](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9687), [@qweeah](https://github.com/qweeah))
- Feat(credential-provider): support k8s service account token ([#8836](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/8836), [@mainred](https://github.com/mainred))

### Bug or Regression

- Chore: bump lbclient to latest API version for AzureStackCloud ([#9308](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9308), [@tyler-lloyd](https://github.com/tyler-lloyd))
- Clean backend pool node destination while reconciling security group ([#9417](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9417), [@clnv](https://github.com/clnv))
- Fix: China cloud endpoints ([#9418](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9418), [@andyzhangx](https://github.com/andyzhangx))
- Fix: Fixes a bug where services sharing an Azure IPv6 pip would not get reconciled due to conflicting ip configurations in the Load Balancer. ([#9596](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9596), [@nilo19](https://github.com/nilo19))
- Fix: Invalidate VMSS cache after VMSS VM changes ([#9876](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9876), [@nilo19](https://github.com/nilo19))
- Fix: invalidate PIP cache on external LB frontend IP config changes ([#9859](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9859), [@Liunardy](https://github.com/Liunardy))
- Fixed the default Helm chart value for `cloudControllerManager.imageRepository` on clusters running Kubernetes 1.31 and earlier. ([#9843](https://github.com/kubernetes-sigs/cloud-provider-azure/pull/9843), [@nojnhuh](https://github.com/nojnhuh))
